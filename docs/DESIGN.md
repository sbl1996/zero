## 1. 修炼与资源

### 1.1 境界与 BP 区间

斗气本源 BP (base power) 是角色修炼进度的核心指标，也决定身体属性与斗气属性的基础数值。
为每个（小）境界定义 BP 区间 [BP_min, BP_max]。进度 p（0–1）线性映射到区间：

```
BP_current = BP_min + (BP_max - BP_min) * p
```

**区间**

| 大境界 | 小境界        | BP区间（含起止）                               | 备注                       |
| --- | ---------- | --------------------------------------- | ------------------------ |
| 一级  | —          | 100–200                                 |                          |
| 二级  | —          | 200–400                                 |                          |
| 三级  | —          | 400–800                                 |                          |
| 四级  | —          | 800–1,600                               |                          |
| 五级  | —          | 1,600–3,200                             |                          |
| 六级  | —          | 3,200–6,400                             |                          |
| 七级  | 初/中/高/巅    | 6,400–12,800（**每段+1,600**）              |                          |
| 八级  | 初/中/高/巅    | 12,800–25,600（**每段+3,200**）             |                          |
| 九级  | 初/中/高/巅    | 25,600–51,200（**每段+6,400**）             |                          |
| 圣域  | 初/中/高/巅/极限 | 51,200–102,400（前四段均分，**极限=区间顶+20%潜力池**） | 极限潜力池只触发部分被动，不线性推高 QiMax |

* 初/中/高/巅表示初阶、中阶、高阶、巅峰四个小境界段

### 1.2 属性

角色包括5项基础属性：
* HP：生命值上限
* QiMax：斗气上限
* ATK：攻击力
* DEF：防御力
* AGI：敏捷度
* REC：恢复力

### 1.3 身体属性与功法

**身体（地基）**：

```
Body = { HP, QiMax, ATK, DEF, AGI, REC }
```

* 大境界提升采用“**先乘后加**”：
  * HP：+20% & +X，X依次为30/70/100/120/130/230/300/340/1000
  * ATK/DEF/AGI：各 +15% & +4/4/2
  * REC：+20% & +5
* **QiMax 与 BP**：`QiMax = 0.5 * BP_current`
* **初始档案**：境界=**一级**（p=0%），`BP≈100`；
  初始 Body 基线：`HP=100, ATK=10, DEF=10, AGI=10, REC=5`（可按职业细化）
  500 GOLD、生命药水×5、斗气药水×3、初始装备：无

### 1.4 斗气属性（Qi_*）与功法侧重

将 BP 转为单位后按功法自动分配（运转期受 `F(t)`）：

```
unit   = BP_current / K_bp2unit        # K_bp2unit = 5

Qi_ATK = unit * coef_ATK
Qi_DEF = unit * coef_DEF
Qi_AGI = unit * coef_AGI
ATK_final = Body_ATK + F(t)*Qi_ATK
DEF_final = Body_DEF + F(t)*Qi_DEF
AGI_final = Body_AGI + F(t)*Qi_AGI
```

### 1.5 修炼、瓶颈与突破

**BP 增长**：

* 冥想每秒钟增长BP
```
ΔBP = K_bp * K_env
K_bp=0.1, K_env：1.0
```

**突破瓶颈**：

* 修炼到当前小境界顶（p=100%）时触发瓶颈，需突破方可继续修炼增长BP。
* 突破瓶颈有多种方式，包括：
  * **强行突破**：清空 Qi，`P = (0.08-0.04p) * f_e * buffs`，单次 ≤35%
    * 成功 → 进下一段，按本章**大境界乘加**改造身体；失败 → 保持瓶颈（强突失败 Qi 清零）

## 2. 斗气资源系统

### 2.1 运转与预热

进入运转→5s 达满 `F(t)`；技能会消耗 Qi，`Qi < 5%*QiMax` 自动退出；重开需重预热。

### 2.2 消耗与恢复

* **动作消耗**（按 QiMax）：见技能表
* **自然恢复**：

```
Qi/sec = (r0 + r1*REC_final) * state_mult * F_mult
r0=1.0, r1=0.06
state_mult：运转=1.0 / 非运转=1.5 / 冥想=3.0
F_mult：运转=F(t) / 非运转&冥想=1.0
```

* **紫焰·恢复被动**：`REC +20`（Body直加，不受F），Qi 恢复×`(1+0.2F)`；圣域极限：+10 REC 与冥想效率 +5%
* **HP 自然恢复**：`HP/sec = 0.1 * REC_final`；战斗中 ×0.5；Boss 场 ×0.25

### 2.3 斗气防御
* 斗气防御在运转斗气后自动生效
* 固定吸收来伤的 60%；**Qi 代价**：吸收值×1.0；Qi 不足则按可吸收量折算后退出

### 2.4 斗气技能
* 斗气技能需要运转斗气方可施放，消耗 Qi，冷却时间独立计算。

### 2.5 斗气闪避
* 斗气闪避是一类特殊技能，消耗 Qi发动，进入闪避状态（窗口），有几率完全规避敌人伤害。
  * 不同闪避技能有不同的窗口、成功率、后摇、冷却与消耗
  * 闪避状态下，伤害免疫成功率取决于双方敏捷比，具体公式见下文

### 2.6 资源回满触发

| 触发        |  HP |  Qi | 说明        |
| --------- | :-: | :-: | --------- |
| **突破成功**  |  ✔  |  ✔  | 过渡到新段后回满  |
| **战败复活**  |  ✔  |  ✔  | 失败结算后返回地图 |
| **胜利**    |  ✖  |  ✖  | 不回满；可直接重赛 |
| **退出/逃跑** |  ✖  |  ✖  | 仅清战斗态，不结算 |

## 3. 怪物系统

### 3.1 核心思路

* 怪物没有直接的属性，而是由隐藏 `BP`与`特化类型`转化而来。
* 怪物没有 QiMax、运转预热、斗气防御、斗气闪避，伤害直接进入 HP 扣减。

### 3.2 从 BP 到属性的换算

#### 特化类型

| 特化             | 说明        | (ATK, DEF, AGI) 系数 |
| -------------- | --------- | ------------------ |
| Balanced（均衡）   | 通用型，攻守兼备  | (0.40, 0.40, 0.20) |
| Attacker（攻击）   | 高输出，低防中速  | (0.55, 0.25, 0.20) |
| Defender（防御）   | 高减伤，输出偏低  | (0.30, 0.55, 0.15) |
| Agile（敏捷）      | 高速、破绽威胁   | (0.35, 0.20, 0.45) |
| Bruiser（重装）    | 压迫近战      | (0.50, 0.40, 0.10) |
| Skirmisher（游击） | 灵活快打      | (0.40, 0.20, 0.40) |
| Mystic（奥术）     | 均衡偏防，技能输出    | (0.35, 0.40, 0.25) |
| Crazy（疯狂）     | 极高攻，极脆    | (0.65, 0.10, 0.25) |

#### 公式与常量

```
unit = BP / K_mon               # K_mon = 5（怪物BP→单位值换算常数）

ATK = round(unit * coef_ATK * S_ATK)
DEF = round(unit * coef_DEF * S_DEF)
AGI = round(unit * coef_AGI * S_AGI)

S_ATK = 1.2
S_DEF = 1.0
S_AGI = 1.0
```

### 3.3 怪物个体差异与精英系统

#### 3.3.1 BP 波动机制 (The BP Flux)

怪物列表中`BP`为基准值`BP_base`，在生成怪物时，基于基础 `BP_base` 引入一个波动系数 `M_flux`。

**核心公式：**
```
# 生成怪物最终BP
BP_final = round(BP_base * M_flux)

# 生成怪物最终HP（HP受体格影响，波动幅度通常略大于BP）
HP_final = round(HP_base * M_flux * M_rank_hp)
```

#### 3.3.2 波动分布模型

采用 **长尾分布** 设计：绝大多数怪物是平庸的，少部分是强壮的，极少数发生“变异”成为精英。

我们定义一个随机数 `R` (0 ~ 100)，根据 R 落在的区间决定怪物的**阶级 (Rank)**。

| 阶级 (Rank) | 出现概率 | 随机数 R区间 | BP 系数 (M_flux) | HP修正 (M_rank_hp) | 韧性 (tough) | 掉落倍率 | 特性描述 |
| :--- | :---: | :---: | :--- | :--- | :--- | :--- | :--- |
| **普通** | 70% | 0~70 | 0.95 ~ 1.05 | 1.0 | 1.0 | 100% | 标准强度的怪物 |
| **强壮** | 20% | 70~90 | 1.06 ~ 1.15 | 1.2 | 1.1 | 150% | 体型略大，攻防明显提升，需认真应对 |
| **精英** | 9% | 90~99 | 1.16 ~ 1.30 | 1.3 | 1.2 | 300% | 属性大幅碾压同级 |
| **灾厄** | 1% | 99~100 | 1.35 ~ 1.60 | 1.5 | 1.3 | 800% | 极度危险！堪比BOSS |

* 掉落倍率影响 GOLD 和 coreDrop 概率（不超过100%）
* TODO: 掉落影响装备掉落率与品质
* BOSS 类怪物默认锁定为 `rank=BOSS` (独立阶级)，不参与普通随机波动

### 3.4 生态连锁反应（TODO）

为了进一步增加趣味性，引入**地图生态参数** `Map_Entropy` (地图熵)。

1. **击杀积累**：玩家在该地图每击杀一只怪物，`Map_Entropy` 增加。
   * 普通：增加1%
   * 强壮：增加3%
   * 精英：增加10%
   * 灾厄：地图熵归零。
2. **熵值阈值**：
   * 当 `Map_Entropy` 超过 90% 时，检查当前整个地图中是否存在灾厄怪物，若存在，触发**"斗气潮汐" (Qi Tide)** 事件，否则地图熵自动回落到0。
3. **斗气潮汐**：
   * 持续 120秒，到时间后地图熵归零，事件结束。
   * 所有普通、强壮、精英怪物消失，只留下灾厄怪物。
   * 击杀灾厄怪物或战斗中逃跑、失败均导致地图熵归零，事件结束。
   * 事件结束时，所有怪物重新刷新。
   * 背景音乐变得急促，环境色调改变。
   * **高风险高回报**：这是玩家刷钱刷装的最佳时刻，也是最容易翻车的时刻。

## 4. 伤害与对抗

### 4.1 参考防御与穿透

* **参考防御常数 K**
  
  由**攻击方威胁等级**取表而来：

  ```
  K = DEF_ref(threat_tier_attacker)
  ```

  * `threat_tier_attacker`：基于**攻击方境界与地图/词缀**综合评定的威胁等级。
  * 暂时仅基于攻击方境界：一到七级依次为25/45/75/127/224/405/750。

* **穿透（由攻击方决定）**

  ```
  PenPct = clamp(0.05 + 0.01*Realm_att, 0, 0.60)
  PenFlat = 来自技能/词缀的固定穿透（默认 0）
  ```

* **tough（目标韧性，入减伤端）**
  玩家=1.0，普通怪视rank而定，BOSS=1.5。

### 4.2 破绽机制

**破绽机制**：敏捷将作为一项**相对属性**，决定攻击方发现并利用敌人弱点的能力。

```
# 1. 计算双方敏捷比
Ratio_AGI = AGI_final_att / AGI_final_def

# 2. 计算最终破绽率 (Chance_Weakness)
Chance_Weakness = 0.05 + 0.75 * clamp(1 - 1/Ratio_AGI, 0, 1)^1.3
## 速查表
| Ratio_AGI | Chance_Weakness |
| ----------|----------------|
| 1.0       | 5%              |
| 1.5       | 23%             |
| 2.0       | 35%             |
| 2.5       | 44%             |
| 3.0       | 49%             |
| 4.0       | 57%             |
| 5.0       | 61%             |
```

### 4.3 斗气闪避判定
* 在闪避窗口内被攻击时，基于双方敏捷比进行判定：

```
# 1. 计算双方敏捷比
Ratio_AGI = AGI_final_att / AGI_final_def

# 2. 计算闪避成功率 (Success)
Success = clamp( base * e^{-k*(Ratio_AGI-threshold)}, 0, 1 )
# 当 Ratio_AGI=1.5 时，Success 开始 < 1.0；Ratio_AGI=2.0 时，Success=80%
base = 1.0, threshold = 1.5, k = log(0.8) / (2.0 - 1.5)

## 速查表
| Ratio_AGI | Success Rate |
| ----------|--------------|
| 1.5       | 100%         |
| 2.0       | 80%          |
| 2.5       | 64%          |
| 3.0       | 51%          |
| 5.0       | 21%          |
| 10.0      | 2%           |
```

### 4.4 结算流程

**1) 命中前判定**
处于闪避窗口且判定成功 → 免伤；否则进入伤害管线。

**2) 物理主干**

```
# 原始伤害（攻击方面板与技能）
raw   = ATK_final * skill.mult * roll(0.8~1.2)

# 有效防御（防守方面板 - 攻击方穿透，再乘以目标韧性）
effDF = max(0, DEF_final - PenFlat) * (1 - PenPct) * tough_target

# 参考防御常数：按“攻击方威胁等级”取表
K     = DEF_ref(threat_tier_attacker)

# 标准化减伤曲线（f = 0.5 的平衡点）
mitig = K / (K + effDF)

# 基础伤害结果（进入斗气防御层前）
dmg   = round(raw * mitig)
```

**3) 破绽判定层**
基于双方的 `AGI_final` 计算破绽率，判定是否触发破绽伤害。
```
Chance_Weakness = F_Weakness(AGI_final_att, AGI_final_def) # 参考 4.2 节公式
IF roll(0,1) < Chance_Weakness:
    dmg = round(dmg * 1.5)  # 破绽伤害，固定+50%
```

**4) 斗气防御层**（若防守方处于“运转”状态则生效）
* 固定吸收来伤的 60%。
* **Qi 代价**：`吸收值 × 1.0`；Qi 不足则按余量吸收后退出运转。
* 余量进入 HP 扣减。

## 5. 技能系统

### 5.1 技能配置

* 战斗界面 4 槽（可重复装，同技能共享冷却）；资源类型：**无消耗 / Qi 消耗**；资源不足不可施放

# 5.2 技能等级

## 5.2.1 基本规则

* 适用对象：除`斗气闪避`外的全部可造成效果的技能（含伤害或明确命中判定的非伤害技）。
* 等级范围：基础为1级，各技能等级上限不同。
* 进度条：每次**使用**获得熟练度 `XP`；**XP 满** → `LV+1`。

## 5.2.2 熟练度获取（每次使用）

```
ΔXP = clamp( ( X0 + B*base_cd ) * (C*hit_bonus) * 0.2, 0, 10 )
# 常量：X0 = 1.0,  B = 0.25,  C = 0.5
# base_cd：技能基础冷却（秒，取技能表“冷却”）
# hit_bonus：若该次使用命中（或生效）=1，否则=0
```

* **反刷衰减**：同一场战斗内，对**同一目标**的第 n 次连续使用：
  `ΔXP *= (0.8)^(n-1)`

## 5.2.3 熟练度上限（每级）

| 等级 (Lv)  | 升级所需 XP (阈值) |
|------------|--------------------|
| **1 → 2**  | **10**             |
| **2 → 3**  | **30**             |
| **3 → 4**  | **80**             |
| **4 → 5**  | **150**            |
| **5 → 6**  | **300**            |
| **6 → 7**  | **500**            |
| **7 → 8**  | **800**            |
| **8 → 9**  | **1200**           |
| **9 → 10** | **2000**           |

## 5.2.4 上限与境界软门槛

```
MaxLV_by_Realm:
  一级~三级：≤3
  四级~六级：≤5
  七级~九级：≤7
  圣域： ≤10
```

* 当角色境界不足时，使用技能不继续积累 `XP`（显示“已达当前上限”）。


### 5.3 斗气功法
* 龙血斗气 (dragon_blood)
  * 基本特性
    * 描述：引动体内沉睡的“青龙”血脉，使血液如奔腾河流般在血管中轰鸣。修炼至深处，体表将覆盖细密的青色龙鳞，肉体力量与防御力呈几何级数暴增，不仅能免疫大部分低阶魔法，更能以纯粹的蛮力粉碎一切阻碍。
    * 属性分配：ATK 0.40 / DEF 0.40 / AGI 0.20
  * 天赋：
    * 【龙战于野】：每消耗一定斗气就会获得可叠加的【血怒】BUFF，增强攻击、防御与斗气恢复。
      * 每消耗 10% QiMax 获得1层【血怒】BUFF，提供+1%攻击、+1%防御与2%斗气恢复加成（独立“加性”乘区），最多叠加20层。

* 金刚斗气 (vajra)
  * 描述：源自“玄武”血脉的苦修法门，通过特殊的呼吸吐纳，将斗气凝练得如同水银般沉重。它并不追求花哨的技巧，而是极致地强化骨骼密度与内脏韧性，让身躯化为战场上不可撼动的移动堡垒，生命气息悠长深厚。
  * 属性分配：ATK 0.45 / DEF 0.50 / AGI 0.05
  * 天赋：
    * 【金刚不坏】：免疫破绽额外伤害，斗气防御显著加强。
      * 破绽机制仍会触发，但因破绽机制对我方造成的额外伤害降低100%。
      * 斗气防御吸收比例提升至80%，Qi 代价降低至吸收值的0.8倍。

* 虎纹斗气 (tiger_stripe)
  * 描述：激发“白虎”血脉中原始的杀戮本能，斗气呈锐利的白金色。该功法极度压榨肌肉纤维的爆发力，使攻击带有天然的撕裂效果。修炼者往往攻势如狂风骤雨，在敌人反应过来之前，便已用武器撕开了对方的喉咙。
  * 属性分配：ATK 0.50 / DEF 0.10 / AGI 0.40
  * 天赋：
    * 【虎煞·绝影】: 闪避成功后获得可叠加的【虎煞】BUFF，提升最终伤害，受到伤害时移除所有层数。
      * 每次闪避成功后获得1层【虎煞】BUFF，提供+4%最终伤害加成（独立“加性”乘区），最多叠加10层。
      * 每层BUFF持续10s，再次闪避成功刷新所有层数持续时间。
      * 受到任何伤害时移除所有层数。

* 紫焰斗气 (purple_flame)
  * 描述：唤醒“朱雀”血脉中的不死特性，将斗气转化为一种拥有奇特治愈力的紫色热流。它能极大程度激活细胞活性，使伤口以肉眼可见的速度愈合。战斗时全身血液如熔岩般灼热，每一次呼吸都在为身体重新注入勃勃生机。
  * 属性分配：ATK 0.30 / DEF 0.30 / AGI 0.15 + **REC 0.25**
  * 天赋：

### 6.3 主属性强化

* `L`为强化等级，`0 ≤ L ≤ 15`。
* 固定加成：
  `FlatBonus(L) = 0 (L=0)`；`L≥1: FlatBonus = round(BaseMain_Q × flatCapMultiplier × FlatProgress(L))`
  默认 `flatCapMultiplier`：武器 25%，盾/铠甲/头盔 18%，其他 15%；`FlatProgress(1→15)=[0.07,0.15,0.22,0.30,0.38,0.48,0.58,0.68,0.78,0.86,0.92,0.96,0.98,0.99,1.00]`
* 百分比加成：
  `MainBonus(L)= 0.01×min(L,5) + 0.02×max(min(L-5,5),0) + 0.03×max(L-10,0)`
* 最终主属性：`FinalMain(L) = round((BaseMain_Q + FlatBonus(L)) × (1 + MainBonus(L)))`
* UI：显示“基础 + 固定 + 百分比 → 总值”，并提示**下一等级**的两项增量。
* 强化区段表:

| 等级段 L | 成功率 | 失败惩罚  |   掉级下限 | 消耗材料   |   消耗金币 |
| ----: | --: | ----- | -----: | ------ | -----: |
|   0–4 | 95% | 无掉级   |      — | 祝福宝石×1 |    500 |
|   5–9 | 80% | 掉 1 级 | **5** | 灵魂宝石×1 |  2,000 |
| 10–14 | 50% | 掉 2 级 | **10** | 奇迹宝石×1 | 10,000 |

* `掉级下限=10` 表示 L≥10 失败后不会低于 10（保留“10级安全锚点”作为分段稳定点）。

* 背包页可穿戴/卸下/丢弃（确认弹窗）；装备页**仅浏览**。

## 7. 战斗系统

### 7.1 流程

进入战斗即启动：玩家即时操作；怪物默认每 **X秒** 行动一次，不同怪物可能有不同的行动频率；任一方 HP≤0 即结算

### 7.2 退出战斗

按钮或切页即可退出；清状态与计时；不结算掉落与 GOLD；**HP/Qi 不回满**

### 7.3 胜败与自动重赛

* **胜利**：`GOLD += rewardGold`；BOSS 首杀解锁下一张野外地图；**0.8s** 后普通怪可自动重赛（**不回满 HP/Qi**）；BOSS 留场展示战利品，点击立绘可重赛（不回满）
* **失败**：`GOLD -= floor(GOLD/3)`；随后 **回满 HP 与 Qi** 并返回地图
* **RNG**：统一 32 位种子；掉落与强化也纳入统一 RNG 序列以复现

### 7.4 消耗品

| ID                              | 名称         | 功能            |                 单价 |
| ------------------------------- | ---------- | ------------- | -----------------: |
| potionHP                        | 生命药水Ⅰ       | 回复 **100 HP** |                 50 |
| potionQi                        | 斗气药水Ⅰ       | 回复 **120 Qi** |                120 |
| potionQiPlus                    | 斗气药水Ⅱ     | 回复 **240 Qi** |                240 |
| coreShardTier1~9                | X级晶核       | 冥想时使用，10s内+BP/s | 100~25,600 |
| blessGem / soulGem / miracleGem | 祝福/灵魂/奇迹宝石 | 强化素材（§6）      | 1000 / 2000 / 5000 |

**战斗中物品**（底部 4 槽）：点击即生效，统一 10s 冷却；不同物品分开计时；默认绑定：生命Ⅰ/斗气Ⅰ/斗气Ⅱ/（空）

### 7.5 晶核系统

**晶核**：修炼加速消耗品，仅在冥想状态下使用。

#### 等级与效果
* **9个等级**：一到九级，对应怪物境界等级
* **加成公式**：`1 × 2^(等级-1)` BP/s（一级+1/s，九级+256/s）
* **持续时间**：固定 **10秒**
* **价格公式**：`100 × 2^(等级-1)` 金币

#### 获取方式
* **战斗掉落**：怪物胜利后概率获得
  * 普通怪物：**20%** 掉落概率
  * 精英怪物：**45%** 掉落概率
  * Boss怪物：**90%** 掉落概率
* **掉落等级**：晶核等级 = 怪物境界等级（一到九级）

#### 使用机制
* **前置条件**：必须处于冥想状态
* **使用效果**：激活后获得额外BP/s加成，10秒后自动失效
* **状态管理**：同时只能激活一个晶核效果，高等级会覆盖低等级

## 9. 任务系统

### 9.1 核心概念

* **任务（Quest）**：由NPC或任务板发布，玩家主动接受与提交。
* **目标（Objective）**：任务的细化条件，详见任务类型。
* **状态（State）**：未解锁 → 可接受 → 已接受 → 可交付（完成未提交）→ 已完成 / 已放弃。

### 9.2 任务生命周期

1. **解锁**
   * 某些任务需要满足前置条件后才可接受。
   * 条件示例：
     * 玩家境界 ≥ 某阈值。
     * 首杀某BOSS后解锁一批新任务。
     * 完成前置任务链后开放后续任务。

2. **接受**
   * 玩家在NPC对话或任务板界面手动点击“接受”。
   * 接受后：
     * 对应目标开始计数。
     * 若为“收集任务物品”，从此刻起开启 **任务物品掉落逻辑**。

3. **进行中**
   * 任务界面显示当前进度。完成或部分完成目标时即时更新。

4. **达成**
   * 所有目标满足 → 状态切换为“可交付”。
   * 玩家需要返回任务来源（或指定交付NPC）提交。

5. **提交/完成**
   * 结算奖励，更新任务链进度，解锁后续任务/地图/商店内容。

6. **放弃（可选）**
   * 玩家手动放弃任务：
     * 清除任务进度。
     * 本阶段简化处理：**已获得的任务物品直接销毁**，防止后续计数混乱。
   * 某些关键任务可以设置为**不可放弃**。

### 9.3 任务类型

* 击杀 X 只 Y 怪物（KillXMonster）
  * 必须在任务被接受之后的击杀才计数。
* 获取 X 个 Y 物品（KillCollectXItem）
  * 任务物品只在任务已接受时有概率掉落。
    * 未接受任务时，怪物不掉该任务物品。
    * 任务完成/放弃后，相关任务物品停止掉落。
  * 任务物品为**非交易、非普通掉落表**的独立类别：
    * 不占用普通掉落槽（避免打乱原有掉落与经济）。
    * 采用“任务叠加掉落层”：判定任务物品之前先走普通掉落。
  * 掉落配置：
    * 任务物品ID：例如“史莱姆凝胶样本 (Slime Gel Sample)”。
    * 掉落怪物集：一般是 1~2 种指定怪物（如仅史莱姆，但也可以“史莱姆或黄金绵羊”）。
    * 基础掉率：例如 40%、60% 等，可随任务等级增加而调低。
    * 每次击杀最多掉落 1 个同类任务物品（防止单次爆出过多秒完任务）。
  * 失败与多次接取
    * 放弃任务时：**自动销毁背包内相关任务物品**。
    * 再次接取时：重新从 0 计数，不保留旧物品。

### 9.4 奖励与难度标记

* **基础奖励构成**：
  * GOLD：覆盖玩家在该任务预期战斗中的药水消耗，并略有盈余。
  * 消耗品：生命药水Ⅰ / 斗气药水Ⅰ/Ⅱ，视难度补一点缓冲。
  * 晶核：作为“修炼加速”的长线奖励，用于鼓励挑战更高阶怪。

* **难度标记**：
  * ★：预计 3–5 分钟内完成的简单任务。
  * ★★：预计 5–10 分钟，或需要进入稍高危险区域。
  * ★★★：地图后段区域或与精英/BOSS相关。

## 10. 地图系统

### 10.1 城市地图

* 城市地图包含非功能节点与功能节点，每个节点用百分比坐标置于底图之上。
* 在非功能节点中，右侧会显示NPC列表，点击NPC名称即可与其对话或接取任务。
* 功能节点包括商店、公会、任务板、驿站等，点击后右侧会出现对应功能，或路由至相应界面。
  * 地图上还有通往其他地图的传送节点，点击后可选择前往目标区域（若已解锁）。

### 10.2 野外地图

* 每张野外地图由一张底图和若干区域节点组成，节点按类型（战斗/功能/传送）组织，节点之间通过连接关系串成明确的路线。
* 节点可以展示当前可选的怪物、驻地 NPC，亦可通过传送节点跨地图流转；玩家在地图界面点击某个区域节点即可移动到该区域并进入战斗或交互。
* 区域节点的状态由进度记录：每个野外地图保留当前所在节点，界面切换时会自动聚焦上一次的区域；传送、解锁等行为会沿着节点图自动调整默认目标。

### 10.3 区域怪物生态

* 区域作为最小的怪物分布单位，每个区域定义一组刷怪权重、批次规模、刷新间隔与重生时间，确保相同的区域总保持一致的怪物组合与强度。
* 怪物列表按权重抽取，BOSS 区域通常只生成单体目标；普通区域同时存在多种怪物，点击即可观察它们的境界、奖励与特化类型，支持手动挑选挑战对象。
* 区域还包括事件 NPC 或地图事件，方便在“区域＝生态+任务”的逻辑下让怪物生态与剧情/资源点共存。

### 10.4 进度与解锁

* 所有城市地图默认可达，第一张野外地图默认解锁，其余野外需依赖相应 BOSS 首杀完成后动态解锁，开新区域会在驿站列表中显示。
* 传送节点提供地图间的出口，穿越前会校验目标区域是否开放；若未开放，会提示“前方区域尚未解锁”，防止误入未准备的高阶区域。
* 刷图与传送结束后，路由会重置到最近一次可用地图，保证玩家返回地图页时落在当前进度可访问的区域之上。

## 12. 调参起点

* **BP→单位**：`K_bp2unit = 5`
* **突破**：强突 `P0=0.08-0.04p`（≤35%）；导师 ×1.15
* **运转**：预热 5s；退出阈值 5% QiMax；成本 1/3/7/12% QiMax；闪避 6%（成功返 4%）
* **恢复**：`Qi/sec=(1+0.06*REC)*state_mult*F_mult`；紫焰×`(1+0.2F)`、`REC+20`；HP 自然恢复 `0.1*REC`（战中×0.5、Boss×0.25）
* **防御**：斗气防御固定吸收来伤的 60%，Qi 代价=吸收×1.0
* **对抗**：`PenPct = clamp(0.05 + 0.01*Realm + 0.03*rank, 0, 0.60)`；`PenFlat` 由技能/装备提供
* **减伤曲线**：平衡点 f=0.5，`K=DEF_ref(realm_stage)`；tough：玩家1.0/怪1.0/BOSS1.5（0.5~2.0 浮动）
